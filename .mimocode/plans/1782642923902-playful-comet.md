# Plan: Add Blog Categories with Filter & Grouping

## Goal

Add a fixed set of 4 categories to blog posts, with:
- Category selector in the admin editor
- Clickable filter tabs on the blog index
- Category-grouped sections on the blog index (like series grouping)
- Category badge on post list items and single post pages

### Categories (fixed set)

| Slug | Display Name |
|------|-------------|
| `people` | The People Behind |
| `anatomy` | Anatomy |
| `footnotes` | Footnotes |
| `deep-currents` | Deep Currents |

## Phase 4: Blog Index (`src/app/blog/page.tsx`)

This is the main UX change. The index will have:

### Filter tabs (client-side, at top of content area)
A row of clickable pill/tab buttons: "All", "The People Behind", "Anatomy", "Footnotes", "Deep Currents"

Since the page is a server component, we need a thin client wrapper for the filter interaction. Approach: keep the server component for data fetching, add a `<BlogFilter>` client component that receives all posts and handles filtering + grouping client-side.

### Category sections
Below the filter tabs, render posts grouped by category (similar to how series are grouped now). Each category section shows:
- Category label as section header
- Posts listed with `BlogPostListItem`

### Implementation approach
1. Server component fetches catalog (includes `categories` group)
2. Pass full data to a new `<BlogIndexContent>` client component
3. Client component manages filter state and renders filtered/grouped posts
