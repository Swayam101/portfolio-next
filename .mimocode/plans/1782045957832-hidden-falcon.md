# Plan: Blog Admin API Documentation + Implementation

## Goal
Create `BLOG_API_DOCS.md` with comprehensive API documentation for a blog admin panel, plus implement the missing endpoints.

## What Exists Today
- `POST /api/blog` — upsert a post (create/update via slug)
- Blog posts stored as YAML strings in MongoDB (`YamlBlogPost` model)
- YAML parsed at render time via `parseBlogYaml`

## New Endpoints to Implement

### 1. `GET /api/blog` — List all posts
- Query params: `search`, `series`, `page`, `limit`
- Returns parsed posts grouped by series + standalone
- For admin dashboard listing

### 2. `GET /api/blog/[slug]` — Get single post (parsed)
- Returns full parsed `BlogPost` JSON
- For post detail view / edit form pre-fill

### 3. `GET /api/blog/raw/[slug]` — Get raw YAML
- Returns raw YAML string + metadata (createdAt, updatedAt)
- For YAML editor in admin panel

### 4. `DELETE /api/blog/[slug]` — Delete post
- Permanently removes a post
- Revalidates cache

### 5. `POST /api/blog/validate` — Validate YAML without saving
- Parses YAML and returns success/error
- For draft validation before publish

### 6. `GET /api/blog/series` — List all series
- Returns unique series with post counts
- For series management UI
